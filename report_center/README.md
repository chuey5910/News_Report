# ศูนย์บันทึกข่าว (Report Center)

ระบบเว็บสำหรับบันทึกและจัดเก็บข้อมูลรายงานข่าว พร้อมระบบสมาชิก (ลงทะเบียน/เข้าสู่ระบบ) และ
บันทึกประวัติการเข้าใช้งานทุกครั้ง เป็น **ระบบแยกต่างหาก** จากเว็บสรุปข่าว RSS เดิม
(`news_report/`, `docs/`) — มี backend + ฐานข้อมูลของตัวเอง (Flask + SQLite) และเปิด
JSON API ให้ระบบเดิมดึงข้อมูลไปแจ้งเตือนได้ (ดูหัวข้อ "การเชื่อมต่อกับระบบแจ้งเตือนเดิม")

## ฟอร์ม "รายงานข่าว" (ฟอร์มเดียว รวมทุกประเภท)

เดิมระบบมี 4 หน้าฟอร์มแยกกัน (ข่าวล่วงหน้า/ปิดข่าว/รายงานสถานการณ์ข่าว/ข่าวทั่วไป) ปัจจุบัน
รวมเหลือ **ฟอร์มเดียว** ชื่อ "รายงานข่าว" — หัวข้อแรกสุดของฟอร์ม (ก่อน "ชื่อกิจกรรม") คือ
**ประเภทรายงาน** ซึ่งเป็น checkbox เลือกได้หลายข้อ:

- ข่าวล่วงหน้า
- ปิดข่าว
- รายงานเหตุการณ์
- ข่าวทั่วไป

ตามด้วยฟิลด์ที่เหลือ: ชื่อกิจกรรม, **ประเภทกิจกรรม** (checkbox: ร้องเรียน/เสวนา/แสดงออกเชิง
สัญลักษณ์ในพื้นที่/ประชุมสมาชิก-รวบรวมกลุ่มจัดกิจกรรม/หน่วยงาน-สส.ลงพื้นที่), **ประเภทกลุ่มปัญหา**
(checkbox: ความมั่นคงสถาบันพระมหากษัตริย์และสังคม/การก่อความไม่สงบและชายแดน/นักวิชาการ-องค์กร
เอกชน-สื่อมวลชน/การเมือง/วิทยาศาสตร์เทคโนโลยีพลังงานสิ่งแวดล้อม/เศรษฐกิจ/ยาเสพติด/ต่างประเทศ),
วันที่+เวลานัดหมาย (ปฏิทิน+เวลาแยกกันตามแบบสากล), วันที่+เวลาสิ้นสุดกิจกรรม, การขออนุญาต (ถ้ามี →
ที่ไหน/กี่วัน), สถานที่นัดหมาย, ชื่อกลุ่ม, **แกนนำ** (เลือกจำนวนคน → เพิ่มช่องชื่อ-นามสกุลตามจำนวน),
จำนวนมวลชน, รูปแบบกิจกรรม, ข้อเรียกร้อง/วัตถุประสงค์, ผู้สนับสนุน, ความเชื่อมโยงกับบุคคล/องค์กรอื่น,
สัมภาระค้างแรม/อุปกรณ์ (มี/ไม่มี → ถ้ามีเพิ่มช่องรายละเอียด), **ยานพาหนะ** (มี/ไม่มี → ถ้ามีเลือก
จำนวนคัน → เพิ่มช่องประเภท/ทะเบียน/จังหวัด/สีตามจำนวน), ข้อมูลน่าสนใจอื่นๆ, แนวโน้ม/ข้อพิจารณา,
ผู้รายงาน, เบอร์ติดต่อ

แกนนำ/ยานพาหนะเก็บในตารางลูกแยกต่างหาก ผูกกับรายงานแต่ละรายการ — เลือกจำนวนจาก dropdown แล้ว
ช่องกรอกจะถูกสร้างขึ้นอัตโนมัติด้วย JavaScript (`static/js/activity-form.js`) ฟิลด์ที่ขึ้นกับเงื่อนไข
(การขออนุญาต/สัมภาระ/ยานพาหนะ) จะซ่อน/แสดงอัตโนมัติ และฟิลด์ที่ถูกซ่อนอยู่จะไม่ถูกนำไปแสดงในหน้าต่าง
ยืนยันก่อนบันทึก — ช่องวันที่ใช้ปฏิทินเลือก (`<input type="date">`) และช่องเวลาใช้ตัวเลือกเวลาของ
เบราว์เซอร์ (`<input type="time">`) โดยระบบจะรวมค่าวันที่+เวลาเป็นค่าเดียวก่อนบันทึกลงฐานข้อมูล

ทุกรายการบันทึกผู้บันทึกและเวลาอัตโนมัติ — ฟอร์มนี้มีไว้สำหรับ**บันทึกข้อมูลเท่านั้น** (ไม่แสดงรายการ
ย้อนหลังในหน้าเดียวกัน) และเมื่อกด "บันทึก" จะขึ้น**หน้าต่างให้ตรวจสอบความถูกต้องของข้อมูลอีกครั้ง**
(สรุปค่าที่กรอกทั้งหมด รวมรายการที่ติ๊กในแต่ละ checkbox group เป็นข้อความเดียว) พร้อมปุ่ม "ยืนยันบันทึก"
/ "แก้ไขข้อมูล" ก่อนจะบันทึกลงฐานข้อมูลจริง

### การเก็บ "ประเภทรายงาน" ลงฐานข้อมูล (พิจารณาความปลอดภัย/ความถูกต้องของข้อมูลแล้ว)

เนื่องจาก "ประเภทรายงาน" มีตัวเลือกคงที่แค่ 4 ค่า (ข่าวล่วงหน้า/ปิดข่าว/รายงานเหตุการณ์/ข่าวทั่วไป)
จึงเก็บเป็น **4 คอลัมน์ boolean แยกกัน** (`is_advance_news`, `is_closure`, `is_incident_report`,
`is_general_news`) แทนการเก็บเป็นข้อความคั่นด้วยจุลภาคแบบเดียวกับ "ประเภทกิจกรรม"/"ประเภทกลุ่มปัญหา"
เพราะ:

- **ปลอดภัยต่อข้อมูลกว่า**: ค่าเป็น boolean ตายตัว ไม่มีความเสี่ยงเรื่องพิมพ์ผิด/ตัวพิมพ์เล็กใหญ่ไม่ตรงกัน
  แบบข้อความอิสระ
- **query/กรองข้อมูลได้เร็วและตรงกว่า**: `WHERE is_closure = true` เร็วกว่าและแม่นยำกว่า
  `WHERE report_type LIKE '%ปิดข่าว%'` (ซึ่งมีโอกาส false positive และทำ index ได้ไม่ดี)
- ฟอร์มยังคงเป็น checkbox กลุ่มเดียวกัน (ใช้ชื่อ input เดียวกันคือ `report_type`) — ฝั่ง view จะ
  แปลงค่าที่ติ๊กไปเป็น 4 คอลัมน์ boolean ตอนบันทึก

ส่วน "ประเภทกิจกรรม"/"ประเภทกลุ่มปัญหา" (ตัวเลือกมากกว่าและเป็นข้อมูลเชิงพรรณนามากกว่าการจัดหมวดหมู่หลัก)
ยังคงเก็บเป็นข้อความคั่นด้วยจุลภาคเหมือนเดิม เพราะใช้งานได้ดีอยู่แล้วและไม่ใช่ประเด็นหลักของฟอร์ม

## ระบบสมาชิกและความปลอดภัย

- **สมัครสมาชิก** ต้องรอ **ผู้ดูแลระบบ (admin) อนุมัติ** ก่อนจึงจะเข้าสู่ระบบได้ (ป้องกันคนนอกเข้าถึงข้อมูล)
- **สิทธิ์ผู้ใช้ 2 ระดับ**: `admin` และ `user`
  - `user` (เจ้าหน้าที่ทั่วไป): เข้าได้เฉพาะหน้า "รายงานข่าว" — ล็อกอินแล้วเข้าหน้านี้ทันที
  - `admin`: เห็นเมนูเพิ่ม 3 หน้า — **ภาพรวม** (สรุปจำนวนแยกตามประเภท/รายงานล่าสุด), **จัดการผู้ใช้**
    (อนุมัติ/ตั้งสิทธิ์), **ประวัติการเข้าสู่ระบบ** — ล็อกอินแล้วเข้าหน้า "ภาพรวม" ทันที — ผู้ใช้ทั่วไป
    ที่พยายามเข้า URL เหล่านี้ตรงๆ จะได้ HTTP 403
- **บันทึกการเข้าสู่ระบบทุกครั้ง** (`login_logs` table) ทั้งที่สำเร็จและล้มเหลว พร้อมเวลา, IP, User-Agent,
  และเหตุผลที่ล้มเหลว (เช่น รหัสผ่านผิด / บัญชียังไม่ได้รับอนุมัติ) — ดูได้ที่เมนู "ประวัติการเข้าสู่ระบบ" (admin เท่านั้น)
- รหัสผ่านเก็บแบบ hash (werkzeug `generate_password_hash`) ไม่เก็บเป็น plain text
- ฟอร์มทุกหน้ามี CSRF protection (Flask-WTF)

## รันในเครื่อง (local)

```bash
cd report_center
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# จากโฟลเดอร์ repo root (พาเรนต์ของ report_center/):
cd ..
python -m report_center.run
# เปิดเบราว์เซอร์ที่ http://127.0.0.1:5001
```

สร้างบัญชีผู้ดูแลระบบคนแรก (จำเป็น เพราะสมาชิกใหม่ต้องรอ admin อนุมัติ):

```bash
export FLASK_APP=report_center
flask create-admin admin "ผู้ดูแลระบบ"
# ระบบจะถามรหัสผ่าน (interactive prompt)
```

## ตัวแปรแวดล้อม (environment variables)

| ตัวแปร | ค่าเริ่มต้น | คำอธิบาย |
|---|---|---|
| `SECRET_KEY` | `dev-secret-key-change-me` | ใช้เซ็น session cookie — **ต้องเปลี่ยนก่อนใช้งานจริง** |
| `DATABASE_URL` | SQLite ที่ `report_center/instance/report_center.db` | เปลี่ยนได้เป็น Postgres/MySQL URL |
| `REPORT_CENTER_API_KEY` | `dev-api-key-change-me` | API key สำหรับให้ระบบภายนอกดึงข้อมูล — **ต้องเปลี่ยนก่อนใช้งานจริง** |

## การเชื่อมต่อกับระบบแจ้งเตือนเดิม (news_report/)

ระบบเดิม (`news_report/`) สามารถดึงรายงานใหม่จากระบบนี้ผ่าน JSON API เพื่อนำไปประกอบการแจ้งเตือน
LINE ได้ (ยังไม่ได้เชื่อมต่ออัตโนมัติในโค้ด — ต้องเพิ่มโค้ดเรียก API นี้ใน `news_report/` เอง):

```
GET /api/reports/latest?since=<ISO8601>&type=<advance|closure|incident|general>&limit=50
Header: X-API-Key: <REPORT_CENTER_API_KEY>
```

`type` เป็น query param เสริม (ไม่ใส่ก็ได้) กรองเฉพาะรายงานที่ติ๊กประเภทนั้น

ตัวอย่าง response:

```json
{
  "count": 1,
  "results": [
    {
      "id": 3,
      "report_type": {"advance": true, "closure": false, "incident": false, "general": false},
      "title": "...",
      "location": "...",
      "event_datetime": "2026-07-09T09:00:00",
      "event_end_datetime": null,
      "demands": "...",
      "leaders": ["..."],
      "vehicles": [{"vehicle_type": "...", "plate_number": "...", "province": "...", "color": "..."}],
      "created_at": "2026-07-09T09:00:00",
      "created_by": "ผู้ดูแลระบบ"
    }
  ]
}
```

## โครงสร้างไฟล์

```
report_center/
  __init__.py      # app factory + CLI (flask create-admin)
  config.py
  extensions.py    # db, login_manager
  models.py        # User, LoginLog, NewsReport (+leader/vehicle child tables)
  forms.py         # NewsReportForm (ฟอร์มเดียว)
  auth.py          # register / login / logout + login audit logging
  reports.py       # dashboard (admin) + news_report (บันทึกรายงาน)
  admin.py         # อนุมัติผู้ใช้ / ดู login logs
  api.py           # JSON API สำหรับระบบภายนอก (API key)
  templates/
  static/css/
  run.py
```
