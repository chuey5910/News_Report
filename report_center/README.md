# ศูนย์บันทึกข่าว (Report Center)

ระบบเว็บสำหรับบันทึกและจัดเก็บข้อมูล 4 ประเภท พร้อมระบบสมาชิก (ลงทะเบียน/เข้าสู่ระบบ) และ
บันทึกประวัติการเข้าใช้งานทุกครั้ง เป็น **ระบบแยกต่างหาก** จากเว็บสรุปข่าว RSS เดิม
(`news_report/`, `docs/`) — มี backend + ฐานข้อมูลของตัวเอง (Flask + SQLite) และเปิด
JSON API ให้ระบบเดิมดึงข้อมูลไปแจ้งเตือนได้ (ดูหัวข้อ "การเชื่อมต่อกับระบบแจ้งเตือนเดิม")

## 4 ประเภทข้อมูล (เมนูซ้าย / ฟอร์มขวา)

| ประเภท | ฟิลด์หลักในฟอร์ม |
|---|---|
| **ข่าวล่วงหน้า** และ **ปิดข่าว** | **ใช้ฟอร์ม/ฟิลด์ชุดเดียวกันทุกประการ** — ชื่อกิจกรรม, **ประเภทกิจกรรม** (checkbox หลายตัวเลือก: ร้องเรียน/เสวนา/แสดงออกเชิงสัญลักษณ์ในพื้นที่/ประชุมสมาชิก-รวบรวมกลุ่มจัดกิจกรรม/หน่วยงาน-สส.ลงพื้นที่), **ประเภทกลุ่มปัญหา** (checkbox หลายตัวเลือก: ความมั่นคงสถาบันพระมหากษัตริย์และสังคม/การก่อความไม่สงบและชายแดน/นักวิชาการ-องค์กรเอกชน-สื่อมวลชน/การเมือง/วิทยาศาสตร์เทคโนโลยีพลังงานสิ่งแวดล้อม/เศรษฐกิจ/ยาเสพติด/ต่างประเทศ), วันที่+เวลานัดหมาย (ปฏิทิน+เวลาแยกกัน), วันที่+เวลาสิ้นสุดกิจกรรม (ปฏิทิน+เวลาแยกกัน), การขออนุญาต (ถ้ามี → ที่ไหน/กี่วัน), สถานที่นัดหมาย, ชื่อกลุ่ม, **แกนนำ (เลือกจำนวนคน → เพิ่มช่องชื่อ-นามสกุลตามจำนวน)**, จำนวนมวลชน, รูปแบบกิจกรรม, ข้อเรียกร้อง/วัตถุประสงค์, ผู้สนับสนุน, ความเชื่อมโยงกับบุคคล/องค์กรอื่น, สัมภาระค้างแรม/อุปกรณ์ (มี/ไม่มี → ถ้ามีเพิ่มช่องรายละเอียด), **ยานพาหนะ (มี/ไม่มี → ถ้ามีเลือกจำนวนคัน → เพิ่มช่องประเภท/ทะเบียน/จังหวัด/สีตามจำนวน)**, ข้อมูลน่าสนใจอื่นๆ, แนวโน้ม/ข้อพิจารณา, ผู้รายงาน, เบอร์ติดต่อ — **ปิดข่าวมีเพิ่ม** ช่อง "ผูกกับข่าวล่วงหน้า (ถ้ามี)" ต่อจากชื่อกิจกรรม |
| **รายงานสถานการณ์ข่าว** และ **ข่าวทั่วไป** | **ใช้หัวข้อ 5W1H ชุดเดียวกันทุกประการ** — หัวข้อ, **Who** (ใคร), **What** (เกิดอะไรขึ้น), **When** (เมื่อไหร่ — ปฏิทิน+เวลา), **Where** (ที่ไหน), **Why** (ทำไม), **How** (อย่างไร) |

ฟิลด์ตัวหนา (แกนนำ/ยานพาหนะ) เก็บในตารางลูกแยกต่างหาก ผูกกับข่าวล่วงหน้า/ปิดข่าวแต่ละรายการ —
เลือกจำนวนจาก dropdown แล้วช่องกรอกจะถูกสร้างขึ้นอัตโนมัติด้วย JavaScript
(`static/js/activity-form.js`, ใช้ร่วมกันทั้งหน้าข่าวล่วงหน้าและปิดข่าว) ฟิลด์ที่ขึ้นกับเงื่อนไข (การขอ
อนุญาต/สัมภาระ/ยานพาหนะ) จะซ่อน/แสดงอัตโนมัติ และฟิลด์ที่ถูกซ่อนอยู่จะไม่ถูกนำไปแสดงในหน้าต่างยืนยัน
ก่อนบันทึก — ช่องวันที่ใช้ปฏิทินเลือก (`<input type="date">`) และช่องเวลาใช้ตัวเลือกเวลาของเบราว์เซอร์
(`<input type="time">`) โดยระบบจะรวมค่าวันที่+เวลาเป็นค่าเดียวก่อนบันทึกลงฐานข้อมูล

ทุกรายการบันทึกผู้บันทึกและเวลาอัตโนมัติ — ทั้ง 4 หน้านี้มีไว้สำหรับ**บันทึกข้อมูลเท่านั้น** (ไม่แสดง
รายการย้อนหลังในหน้าเดียวกัน) และเมื่อกด "บันทึก" จะขึ้น**หน้าต่างให้ตรวจสอบความถูกต้องของข้อมูลอีกครั้ง**
(สรุปค่าที่กรอกทั้งหมด) พร้อมปุ่ม "ยืนยันบันทึก" / "แก้ไขข้อมูล" ก่อนจะบันทึกลงฐานข้อมูลจริง

## ระบบสมาชิกและความปลอดภัย

- **สมัครสมาชิก** ต้องรอ **ผู้ดูแลระบบ (admin) อนุมัติ** ก่อนจึงจะเข้าสู่ระบบได้ (ป้องกันคนนอกเข้าถึงข้อมูล)
- **สิทธิ์ผู้ใช้ 2 ระดับ**: `admin` และ `user`
  - `user` (เจ้าหน้าที่ทั่วไป): เข้าได้เฉพาะ 4 หน้าบันทึกข่าว — ล็อกอินแล้วเข้าหน้า "ข่าวล่วงหน้า" ทันที
  - `admin`: เห็นเมนูเพิ่ม 3 หน้า — **ภาพรวม** (สรุป/รายการล่าสุดทุกประเภท), **จัดการผู้ใช้** (อนุมัติ/ตั้งสิทธิ์),
    **ประวัติการเข้าสู่ระบบ** — ล็อกอินแล้วเข้าหน้า "ภาพรวม" ทันที — ผู้ใช้ทั่วไปที่พยายามเข้า URL เหล่านี้ตรงๆ
    จะได้ HTTP 403
- **บันทึกการเข้าสู่ระบบทุกครั้ง** (`login_logs` table) ทั้งที่สำเร็จและล้มเหลว พร้อมเวลา, IP, User-Agent,
  และเหตุผลที่ล้มเหลว (เช่น รหัสผ่านผิด / บัญชียังไม่ได้รับอนุมัติ) — ดูได้ที่เมนู "ประวัติการเข้าสู่ระบบ" (admin เท่านั้น)
- รหัสผ่านเก็บแบบ hash (werkzeug `generate_password_hash`) ไม่เก็บเป็น plain text
- ฟอร์มทุกหน้ามี CSRF protection (Flask-WTF)

## รันในเครื่อง (local)

```bash
cd report_center
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# จากโฟลเดอร์ repo root (พาเรนต์ของ report_center/):
cd ..
python -m report_center.run
# เปิดเบราว์เซอร์ที่ http://127.0.0.1:5001
```

สร้างบัญชีผู้ดูแลระบบคนแรก (จำเป็น เพราะสมาชิกใหม่ต้องรอ admin อนุมัติ):

```bash
export FLASK_APP=report_center
flask create-admin admin "ผู้ดูแลระบบ"
# ระบบจะถามรหัสผ่าน (interactive prompt)
```

## ตัวแปรแวดล้อม (environment variables)

| ตัวแปร | ค่าเริ่มต้น | คำอธิบาย |
|---|---|---|
| `SECRET_KEY` | `dev-secret-key-change-me` | ใช้เซ็น session cookie — **ต้องเปลี่ยนก่อนใช้งานจริง** |
| `DATABASE_URL` | SQLite ที่ `report_center/instance/report_center.db` | เปลี่ยนได้เป็น Postgres/MySQL URL |
| `REPORT_CENTER_API_KEY` | `dev-api-key-change-me` | API key สำหรับให้ระบบภายนอกดึงข้อมูล — **ต้องเปลี่ยนก่อนใช้งานจริง** |

## การเชื่อมต่อกับระบบแจ้งเตือนเดิม (news_report/)

ระบบเดิม (`news_report/`) สามารถดึงรายงานใหม่จากระบบนี้ผ่าน JSON API เพื่อนำไปประกอบการแจ้งเตือน
LINE ได้ (ยังไม่ได้เชื่อมต่ออัตโนมัติในโค้ด — ต้องเพิ่มโค้ดเรียก API นี้ใน `news_report/` เอง):

```
GET /api/reports/latest?since=<ISO8601>&category=<advance|closure|situation|general>&limit=50
Header: X-API-Key: <REPORT_CENTER_API_KEY>
```

ตัวอย่าง response:

```json
{
  "count": 2,
  "results": [
    {
      "category": "advance",
      "category_label": "ข่าวล่วงหน้า",
      "id": 3,
      "title": "...",
      "detail": "...",
      "created_at": "2026-07-09T09:00:00",
      "created_by": "ผู้ดูแลระบบ"
    }
  ]
}
```

## โครงสร้างไฟล์

```
report_center/
  __init__.py      # app factory + CLI (flask create-admin)
  config.py
  extensions.py    # db, login_manager
  models.py        # User, LoginLog, AdvanceNews/NewsClosure (+leader/vehicle), SituationReport/GeneralNews (5W1H)
  forms.py         # WTForms ต่อฟอร์มแต่ละประเภท
  auth.py          # register / login / logout + login audit logging
  reports.py       # list + create ต่อ 4 ประเภท
  admin.py         # อนุมัติผู้ใช้ / ดู login logs
  api.py           # JSON API สำหรับระบบภายนอก (API key)
  templates/
  static/css/
  run.py
```
