# News_Report

ระบบดึงข่าวจาก RSS (ทุกสำนักข่าว ทั้งไทยและต่างประเทศ) มากรองเฉพาะข่าวที่เกี่ยวข้องกับ
17 จังหวัดภาคเหนือ/เหนือตอนล่าง สรุปแบบง่าย (ไม่ใช้ AI) พร้อมอ้างอิงแหล่งข่าวต้นฉบับ
เผยแพร่เป็น**เว็บแอปสรุปข่าว** (GitHub Pages, ไม่มี backend) ที่กรอง/ค้นหาได้จริง — เลือกดูตาม
จังหวัด, สำนักข่าว, ในประเทศ/ต่างประเทศ, ค้นหาคำในหัวข้อ/เนื้อหา พร้อมกราฟสถิติจำนวนข่าวต่อจังหวัด
ในแต่ละจังหวัดจะจัดกลุ่มตามสำนักข่าว (สื่อในประเทศขึ้นก่อนสื่อต่างประเทศเสมอ) — และแจ้งเตือนผ่าน
LINE Official Account ว่ามีสรุปข่าวใหม่พร้อมลิงก์ไปอ่าน วันละ 2 รอบ (07:00 น. และ 16:00 น.)
โดยอัตโนมัติผ่าน GitHub Actions — รอบ 16:00 น. จะรายงานเฉพาะข่าวใหม่ที่ยังไม่เคยรายงานไปตั้งแต่
รอบ 07:00 น. ของวันเดียวกัน (ดูหัวข้อ "การรันวันละ 2 รอบ" ด้านล่าง)

รายละเอียดการออกแบบทั้งหมดอยู่ที่ [`docs/FEATURE_X_PLAN.md`](docs/FEATURE_X_PLAN.md)

## จังหวัดเป้าหมาย (17 จังหวัด)
เชียงใหม่, เชียงราย, ลำพูน, ลำปาง, พะเยา, แพร่, น่าน, แม่ฮ่องสอน, ตาก,
สุโขทัย, อุตรดิตถ์, กำแพงเพชร, พิษณุโลก, พิจิตร, เพชรบูรณ์, นครสวรรค์, อุทัยธานี

แก้ไขรายชื่อจังหวัด/คำพ้องได้ที่ `config/provinces.yaml`
แก้ไขรายชื่อ RSS feed ได้ที่ `config/feeds.yaml` (ไม่ต้องแก้โค้ด)

## รันในเครื่อง (local)
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements-dev.txt

# รันทดสอบหน่วย (ไม่ต้องต่อเน็ต, ไม่ต้องมี LINE token)
pytest

# รัน pipeline จริง (ต้องต่อเน็ตเพื่อดึง RSS จริง)
python -m news_report.main
```

ผลลัพธ์จะถูกเขียนไปที่:
- `data/seen.db` — กันข่าวซ้ำระหว่างรอบ
- `data/reports/<YYYY-MM-DD>.json` — ผลสรุปของแต่ละวัน จัดกลุ่มตามจังหวัด (ต้นฉบับ/แหล่งข้อมูลจริง)
- `docs/` — เว็บแอป (สำหรับ GitHub Pages): `docs/index.html` เป็นแอปหน้าเดียว (vanilla JS, ไม่มี
  build step/dependency ภายนอก) ที่ดึงข้อมูลจาก `docs/data/<date>.json` และ `docs/data/index.json`
  (รายชื่อวันที่ทั้งหมด) มา filter/ค้นหา/แสดงกราฟฝั่ง client ทั้งหมด

ถ้ายังไม่ตั้งค่า `LINE_CHANNEL_ACCESS_TOKEN` ขั้นตอนแจ้งเตือน LINE จะ log error แล้วข้ามไป
โดยไม่ทำให้ pipeline ล้มเหลว (ไฟล์รายงาน/เว็บยังถูกสร้างตามปกติ)

## ตั้งค่าเพื่อรันอัตโนมัติบน GitHub
1. **เปิด GitHub Pages**: Settings → Pages → Source: Deploy from a branch → เลือก branch `main`
   + โฟลเดอร์ `/docs` → กด Save แล้ว GitHub จะโชว์ URL ของเว็บ (เช่น
   `https://<user>.github.io/News_Report`) ที่ด้านบนของหน้า Pages settings นี้เอง
2. **ตั้งค่า Secret**: Settings → Secrets and variables → Actions → New repository secret
   - `LINE_CHANNEL_ACCESS_TOKEN` — Channel access token (long-lived) จาก LINE Developers Console
     ของ LINE OA ที่ต้องการ broadcast (ดูวิธีเอา token ในแชทที่คุยกันไว้)
3. **ตั้งค่า Variable (ไม่ใช่ secret เพราะไม่ใช่ข้อมูลลับ)**: Settings → Secrets and variables →
   Actions → Variables → New repository variable
   - `SITE_BASE_URL` — URL ของ GitHub Pages เช่น `https://<user>.github.io/News_Report`
     (ไม่ต้องมี `/` ต่อท้าย) — **สำคัญ**: ถ้าไม่ตั้งค่านี้ ข้อความ LINE จะไม่มีปุ่ม/ลิงก์ไปหน้าเว็บเลย
     (แค่ประกาศว่ามีสรุปข่าวเฉยๆ) เพราะไม่รู้ว่าจะลิงก์ไปที่ไหน
4. **รันครั้งแรกด้วยมือ** เพื่อ seed เว็บเพจตั้งต้น: Actions → Daily News Report → Run workflow
   (ก่อนถึงรอบ cron เว็บ Pages จะยังไม่มีอะไรให้แสดง)

หลังจากนั้นระบบจะรันอัตโนมัติวันละ 2 รอบเวลาไทย **07:00 น.** และ **16:00 น.**
(`.github/workflows/daily-report.yml`)

## การรันวันละ 2 รอบ (ไม่รายงานข่าวซ้ำ)
- `data/seen.db` เก็บ guid ของทุกข่าวที่เคย**ตรงกับ 17 จังหวัด**และถูกรายงานไปแล้ว ไม่ว่าจะรอบ
  07:00 น. หรือ 16:00 น. — รอบถัดไปในวันเดียวกันจะดึงข่าวใหม่ทั้งหมดมาเช็คอีกครั้ง แต่ข่าวที่ guid
  ซ้ำกับที่เคยรายงานไปแล้วจะถูกข้ามอัตโนมัติ
- `data/reports/<YYYY-MM-DD>.json` (และหน้าเว็บของวันนั้น) จะ**สะสม**ผลจากทั้ง 2 รอบเข้าด้วยกัน
  ไม่ใช่เขียนทับ — รอบ 16:00 น. จะเพิ่มเฉพาะข่าวใหม่เข้าไปในหน้าเดิมของวันนั้น
- ข้อความแจ้งเตือน LINE ของแต่ละรอบจะสรุปเฉพาะข่าว**ใหม่ของรอบนั้น**เท่านั้น (ไม่ใช่ยอดรวมทั้งวัน)
  และ**ถ้ารอบนั้นไม่มีข่าวใหม่เลย ระบบจะข้ามการส่ง LINE ไปเลย** (เว็บ/รายงานยังอัปเดตตามปกติ)

## เว็บแอป (docs/index.html)
- **Filter**: จังหวัด, สำนักข่าว, ในประเทศ/ต่างประเทศ — เลือกได้พร้อมกันหลายตัว
- **ค้นหา**: พิมพ์คำค้นในหัวข้อ/เนื้อหาข่าว (client-side, ไม่ต้องต่อเน็ตเพิ่ม)
- **กราฟ/สถิติ**: แถบแสดงจำนวนข่าวต่อจังหวัด อัปเดตตาม filter ที่เลือกแบบเรียลไทม์
- **เลือกวันที่**: dropdown ด้านบนสลับดูรายงานย้อนหลังของวันอื่นได้ (โหลด JSON ของวันนั้นเฉพาะตอนเลือก)
- **"ข่าวทั่วไปวันนี้"**: เมนูปักหมุดด้านซ้าย (เหนือรายชื่อจังหวัด) — ข่าวที่ดึงมาแล้วแต่**ไม่ตรงกับ
  17 จังหวัดเลย** จะไม่ถูกตัดทิ้ง แต่ย้ายมารวมไว้ที่นี่แทน เพื่อไม่ให้พลาดข่าวที่น่าสนใจ แม้จะไม่เกี่ยว
  กับพื้นที่เป้าหมาย
- **🔥 badge "ข่าวใหญ่"**: `news_report/trending.py` ตรวจจับข่าวที่**หลายสำนักข่าวรายงานพร้อมกันด้วย
  หัวข้อคล้ายกันมาก** (เทียบความคล้ายของหัวข้อข่าวแบบ character n-gram — ไม่ใช้ AI/LLM) ถ้ามีตั้งแต่
  3 สำนักข่าวขึ้นไปรายงานเรื่องเดียวกัน จะติด badge นี้บนข่าวนั้นทุกที่ที่ปรากฏ (ทั้งหน้าจังหวัดและ
  ข่าวทั่วไป) พร้อม tooltip บอกจำนวนสำนักข่าว — หมายเหตุ: เทียบความคล้ายเฉพาะข่าวในรอบเดียวกัน
  (07:00 หรือ 16:00) ยังไม่เทียบข้ามรอบของวันเดียวกัน
- **ไม่มี backend**: ทุกอย่างรันในเบราว์เซอร์ ข้อมูลมาจากไฟล์ JSON static ที่ generate ไว้ล่วงหน้า
  จึงยังคง host ฟรีบน GitHub Pages ได้เหมือนเดิม
- **หมายเหตุเรื่องเนื้อหาสรุป**: สรุปยังคงเป็นข้อความจากต้นฉบับ (title/description ของ RSS) ไม่ใช่
  การวิเคราะห์แบบ 5W1H ที่แยกฟิลด์ Who/What/When/Where/Why/How ให้อัตโนมัติ เพราะการแยกฟิลด์แบบนั้น
  ต้องใช้ AI/LLM ช่วยตีความภาษา (ซึ่งตอนนี้เลือกไม่ใช้) — ถ้าต้องการฟีเจอร์นี้ในอนาคต ต้องเพิ่มการ
  เรียก Claude API ต่อข่าว (มีค่าใช้จ่ายต่อการเรียก)

## ระบบบันทึกข่าว (report_center/) — ระบบแยกต่างหาก

นอกจากเว็บสรุปข่าว RSS ด้านบน (static, ไม่มี backend) repo นี้ยังมี **`report_center/`** ซึ่งเป็น
เว็บแอปแยกต่างหาก (Flask + SQLite, มี backend/ฐานข้อมูลของตัวเอง) สำหรับให้เจ้าหน้าที่บันทึกข้อมูล
4 ประเภท: **ข่าวล่วงหน้า / ปิดข่าว / รายงานสถานการณ์ข่าว / ข่าวทั่วไป** พร้อมระบบสมัครสมาชิก+เข้าสู่ระบบ
(ต้องรอ admin อนุมัติ) และบันทึกประวัติการเข้าใช้งานทุกครั้ง — เปิด JSON API (มี API key) ให้ pipeline
ข่าว RSS เดิมดึงไปใช้แจ้งเตือนได้ในอนาคต ดูรายละเอียดทั้งหมดที่ [`report_center/README.md`](report_center/README.md)

## หมายเหตุสำคัญ
- รายชื่อ RSS feed ใน `config/feeds.yaml` เป็นตัวอย่างเริ่มต้น **ควรตรวจสอบว่า URL ยังใช้งานได้จริง**
  ก่อนพึ่งพาใช้งานจริง เพราะ feed อาจเปลี่ยน URL หรือปิดตัวได้โดยไม่แจ้งล่วงหน้า
- การแปลข่าวต่างประเทศใช้ `deep-translator` (ครอบ Google Translate แบบไม่เป็นทางการ ไม่ต้องขอ
  API key) จึงไม่มี SLA รับประกันความเสถียร มี retry ในตัว แต่ถ้าพบปัญหาใช้งานไม่ได้บ่อยในระยะยาว
  ให้พิจารณาเปลี่ยนไปใช้ DeepL หรือ Google Cloud Translation API แทน
- LINE Notify ถูกยกเลิกไปแล้วตั้งแต่ 31 มี.ค. 2025 ระบบนี้ใช้ **LINE Messaging API**
  (Broadcast Message) ของ LINE OA แทน
